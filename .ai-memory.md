# AI Memory — Service Manager (Electron)

## Tech Stack
- Electron 28, Node.js, vanilla JS (no bundler)
- nodeIntegration: true, contextIsolation: false (renderer dùng require trực tiếp)
- electron-builder (NSIS target, Windows)

## Project Structure (sau refactor)
```
src/
  main/
    index.js       ← entry point Electron (createWindow, lifecycle, shortcuts)
    ipc.js         ← tất cả ipcMain.on handlers; SERVICE_CONFIGS để thêm service mới
    services.js    ← spawn/stop/update/getStatus; processes registry
    settings.js    ← load/save settings.json từ userData
    tray.js        ← Tray create/updateMenu/notify
  renderer/
    index.js       ← entry point renderer (init tất cả modules)
    state.js       ← runtime state { router, openclaw }
    ui.js          ← DOM helpers: setStatus, addLog, toast, metrics, alert, ripple...
    views/
      dashboard.js ← service cards, IPC handlers cho status/log/update
      logs.js      ← clear all logs
      settings.js  ← load/save settings UI
    components/
      logPanel.js       ← collapsible log + filter
      commandPalette.js ← Ctrl+K palette, extensible qua addCommand()
index.html         ← trỏ src/renderer/index.js
package.json       ← main: src/main/index.js
```

## Patterns
- SERVICE_CONFIGS trong ipc.js: thêm service mới chỉ cần thêm 1 entry
- commandPalette.addCommand() để đăng ký lệnh mới từ module khác
- Mỗi view là module độc lập export { init }
- ui.js là pure DOM helpers, không có state
- **Renderer require path**: index.html dùng `require(path.join(__dirname, 'src/renderer/index.js'))` để __dirname trong các module con resolve đúng
- **Settings**: settings object có `_save()` và `_settingsPath` (không serialize vào JSON), chỉ save các key trong DEFAULTS + windowBounds

## Tools đã có
- **Hẹn giờ tắt máy** (`src/renderer/views/tools/shutdown.js` + `src/main/tools/shutdown.js`)
  - Tắt máy / khởi động lại sau thời gian tùy chọn
  - Countdown realtime + progress bar đổi màu (xanh → vàng → đỏ)
  - Preset nhanh: 5m, 10m, 15m, 30m, 1h, 2h
  - Hủy lệnh bất kỳ lúc nào
  - IPC: shutdown-schedule, shutdown-cancel, shutdown-now

- **Reset IDM Trial** (`src/renderer/views/tools/idm-reset.js` + `src/main/tools/idm-reset.js`)
  - Reset thời gian dùng thử của Internet Download Manager
  - Tự động kill IDM processes (IDMan.exe, IEMonitor.exe)
  - Xóa registry keys: ConfigTime, MData
  - Reset giá trị Thread=1, Model=0x68
  - Xóa folder %appdata%\IDM
  - Kiểm tra trạng thái IDM đang chạy hay không
  - IPC: idm-reset-trial (handle), idm-check-running (handle)

## Bug Fixes
- Button ripple no longer changes button text color on click.
  - Root cause: `.ripple` used `mix-blend-mode: overlay`, so the ripple blended with foreground text instead of only the button background.
  - Fix: removed the ripple blend mode, isolated `.btn`, and wrapped button content in `.btn-content` so text/icons stay above the ripple layer.
  - Files touched: `style.css`, `src/renderer/ui.js`
  - Remaining risk: future CSS that targets direct button children should account for the `.btn-content` wrapper.
- Native button active/focus styling could still override button text color during press-and-hold.
  - Root cause: `.btn` and `.btn-icon` still relied on browser default button appearance, so Chromium could apply native pressed/focus painting and text-selection highlight on top of custom styles.
  - Fix: reset appearance for `.btn`/`.btn-icon`, locked foreground color across `:active/:focus/:focus-visible`, disabled text selection for button content, and kept ripple constrained below the content layer.
  - Files touched: `style.css`, `src/renderer/ui.js`
  - Remaining risk: if new button variants are added, they should define `--btn-fg` so active/focus color locking stays correct.
- Rolled back the temporary System Metrics feature, including the dedicated renderer view, navigation tab, IPC endpoint, and service-side CPU/RAM sampling code.

- **IDM Reset buttons không hoạt động khi click**
  - Symptom: Khi ấn nút "Kiểm tra IDM" hoặc "Reset IDM Trial" không có phản hồi gì
  - Root cause: File `src/renderer/views/tools/idm-reset.js` gọi sai tên hàm từ `ui.js`:
    - Gọi `ui.toast()` nhưng thực tế là `ui.showToast()`
    - Gọi `ui.addRipple()` nhưng hàm này không tồn tại (chỉ có `ui.initRipple()` để init global)
  - Fix: Sửa tất cả `ui.toast()` → `ui.showToast()` và xóa các lệnh gọi `ui.addRipple()` (ripple effect đã được init global)
  - Files touched: `src/renderer/views/tools/idm-reset.js`
  - Lesson learned: Luôn kiểm tra exports của module trước khi sử dụng. Ripple effect trong dự án này được init global qua `ui.initRipple()`, không cần gọi thủ công cho từng button.
  - Remaining risk: Các tool mới cần chú ý sử dụng đúng tên hàm từ `ui.js` exports
